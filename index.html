OptiTrip: Detecting Falls Using Optical Flow

Fall detection is an active area of research that has major potential impact. In the US, the aging population is constantly growing. Along with this growth, the fall death rate is increasing - up 30% from 2007 to 2016. Every year, over 3 million elderly people are treated in emergency departments as a result of fall injuries, and one in five falls causes severe injury, such as broken bone or head injury. Accurate detection of falls is vital to timely intervention. Current state-ofthe-art focuses on video- and sensor-based detection. Video based monitoring can provide critical support for detection of injury and falls, enabling improved response times and mitigating long term effects. We focused our project scope in fall detection using video due to the ubiquitous video monitoring in hospitals and elderly care homes where such applications can be most helpful. We aim to use optical flow and human pose tracking to localize falls from standard RGB video feed. We performed our analysis with videos from a standard falls database labeled with fall onset and termination, and our own collected data with simulated falls. While we specifically mention the impact to older populations, accurate fall detection can extend to other vulnerable populations including people affected by seizures, people with limited mobility, and patients recovering in hospital.

Autonomous Rideshare Planner and Controller

Project Report

This project focuses on solving the dynamic ride sharing problem which was defined as the problem in which the following conditions exist:

  1. A random start position is known
  2. There are four randomly located waypoints that each need to be visited only once and in no specified order
  3. There is one final goal that does not change which is centralized given the positional bounds of the problem
  4. All dynamic constraints are known (including location of obstacles)
  5. The goal is to minimize the control effort (the smoothness of the ride) and keep the controlled vehicle as close to the ideal path along the route

Solving the problem with random orientations of stopping position is too complex so the states of each stops are simplified. Dijkstra’s algorithm and dynamic programming together provide a valid solution to minimize the distance between all the way points. Multiple shooting solves the minimal control effort and linear velocity between each stop. A nonlinear controller designed by feedback linearization can track the desired trajectory. Combining these three subproblems, we optimize the dynamic ride sharing problem with minimum distance and minimum control effort

UR5 Robot Manipulation Using Inverse Kinematics, Resolved Rate, and Gradient Control Methods

Project Report

For the final project, we had the goal of implementing a movement sequence on the ur5 robot using an inverse kinematics based, differential kinematics based, and gradientdescent based control method. The movement sequenecs followed this general format:

  1. Prompt user to move robot to some user chosen home configuration - it is recommended thatthis be half-way between the two desired positions
  2. User presses mouse buttons to record the home position joint angles
  3. Prompt user to move robot to specified start configuration
  4. User presses mouse buttons to record the start position joint angles
  5. Prompt user to move robot to specified target configuration
  6. User presses mouse buttons to record the target position joint angles
  7. Move ur5 robot to home configuration
  8. Compute an appropriate gesture move based on which desired configuration is input as the start position
  9. Move robot to gesture position to indicate which cup the robot will be moving towards first
  10. Prompt user for the control scheme requested
  11. User enters value corresponding to desired control scheme
  12. Control scheme specified is executed with user input home, start, and target configurations

Design of Custom Low-Cost 3D-Printed Parallel Gripper with Force Control

Project Report

This project focused on producing a gripper with comparable grasping reach to the Robotiq 3-finger gripper but that was more compact in space for more possible variety in grasp approach for deformable objects. Several iterative prototypes were designed, manufactured, and tested. One of the prototypes were selected for further developmenet. The hardware was iterated on and eventually moved over to being majority 3D printed. The low cost of the non-3D printed components and ease of replacement of 3D printed components made the manipulator ideal for testing riskier planning algorithms where collision had more potential to occur or with objects where the gripper needed to be closer to the surface the object was on. Testing of the manipulator allowed to determine and prove a reliable gripper lifecycle for the expected test demands of the lab. Force sensing using basic presure sensors at the fingertips was added to the manipulator to further mimic the capabilities of the off-the-shelf manipulator and make the gripper more robust to grasping a variety of objects.

Senior Design Project: Reduced Turning Radius Solution for an Off-Road Vehicle

Project Report

This project focused on finding a unique approach to the desire of reducing an off-road vehicles turning radius while meeting the following demands:

  1. has no significant affect on the vehicle’s rear-suspension geometry
  2. has low rotational inertia
  3. does not distract the driver from steering, accelerating, and braking
  4. does not result in a loss of rear wheel traction
  5. is durable enough to last extreme driving conditions (including weather) and over 50 hours of continous driving
  6. does not exceed a budget of $1,000

To meet these demands and also have a significant impact in reducing the low-speed turning radius of the vehicle, our team selected to design and prototype a design which added freewheel clutched to the single output shaft to allow for reduce frictional torque on the vehicle when turning at low speed thus resulting in an increased turn radius.
